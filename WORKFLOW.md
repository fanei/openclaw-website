# 开发工作流程约定

## 核心原则

**所有开发工作必须通过Coding Agent完成。**

## 我的工作范围

### ✅ 我的职责（安排和监控）

1. **启动Coding Agent**
   - 根据任务描述启动合适的agent（Codex、Claude Code、Pi等）
   - 提供清晰的任务描述和技术要求

2. **监测任务进度**
   - 定期检查agent的状态
   - 查看执行日志
   - 验证输出结果

3. **验证完成情况**
   - 确认代码是否按预期工作
   - 检查是否需要修改
   - 向用户汇报最终结果

4. **沟通和协调**
   - 向用户报告进度
   - 传递agent的反馈
   - 协助解决遇到的问题

### ❌ 不应该做的

- ❌ 直接编写项目代码
- ❌ 直接创建/编辑项目文件（除非是简单的配置）
- ❌ 直接执行开发命令（npm install、git push等，除非是监控）
- ❌ 试图绕过coding agent完成开发任务

## Coding Agent工作区

**开发项目目录**: `~/.openclaw/workspace-coding/`

所有开发工作应该在这个工作区内进行，保持项目集中管理。

## 任务示例

### 正确方式

**用户**: "帮我添加一个新功能..."

**我的做法**:
1. 启动coding agent
2. 传递任务描述
3. 监测执行过程
4. 验证结果
5. 向用户汇报

### 错误方式

**用户**: "帮我添加一个新功能..."

**我的做法**:
❌ 直接编辑文件
❌ 直接运行命令

## 例外情况

以下情况我**可以**直接操作：

1. **简单的配置更改**
   - 修改环境变量
   - 更新配置文件（非项目源代码）

2. **监控和检查**
   - 查看文件内容
   - 检查git状态
   - 运行测试命令（用于验证）

3. **文档和非代码工作**
   - 创建文档
   - 写说明
   - 组织文件

4. **紧急修复**
   - 但需要先告知用户，解释原因

## 工作流程模板

```
用户请求 → 我启动coding agent → agent执行开发 → 我监测进度 → 验证完成 → 向用户汇报
```
